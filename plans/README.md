## Findings considered and rejected

(So they are not re-audited next run.)

- **CORS wildcard as a CSRF risk** — the backend has no cookie/session/credential
  auth, so there is no ambient authority for a cross-site request to abuse. `AllowOrigins: "*"`
  is appropriate for a public read API; the write endpoints are unauthenticated
  by design (proof-of-control is moving into the request body via plan 008, not
  cookies). Not a CSRF vulnerability.
- **Scalar API docs bloating the client bundle** — `app/docs/route.ts` is a
  route handler (`export const GET = ApiReference({...})`), not a page component,
  so `@scalar/nextjs-api-reference` is not in the app's client bundle. No action.
- **@bsv/sdk "<2.0.0" advisory on the direct dependency** — the audit phrasing
  was misleading: the project's direct `@bsv/sdk` is `2.1.4` (patched). Only the
  older transitive copies under `@1sat/*` are affected; tracked in plan 011, not
  a first-party issue.
- **`SetAvatar` non-atomic 3-write sequence** (current → feed → meta) — real but
  low impact: requires a mid-sequence Redis failure, and the feed self-heals on
  the next write (newest-wins) and the reconciler. Not planned now; revisit if
  Redis flakiness is observed.
- **`upload-dialog.tsx` god component (~725 lines)** — real maintainability debt
  but MED-risk refactor with low functional payoff; deferred. Revisit if that
  component sees frequent change.
